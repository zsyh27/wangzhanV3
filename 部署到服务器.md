# 部署到腾讯云服务器指南

本文档介绍如何将项目部署到腾讯云Ubuntu服务器（目录：`/root/wangzhan/`）。

## 前置条件

- 腾讯云服务器（Ubuntu系统）
- 已安装Git
- 已配置好域名解析（将域名指向服务器公网IP）
- 已在服务器 `/root/wangzhan/` 目录下添加 `baidu_verify_codeva-bMl4jP7kZ0.html` 验证文件

## 部署流程

### 第一步：在GitHub创建仓库

1. 登录 GitHub (https://github.com)
2. 点击右上角 "+" → "New repository"
3. 填写信息：
   - Repository name: `wangzhan`
   - Description: `暖通行业网站 - 湖北科信达机电设备有限公司`
   - 选择 "Public" 或 "Private"
   - **不要勾选** "Add a README file"（因为我们已经有代码了）
4. 点击 "Create repository"

### 第二步：推送本地代码到GitHub

在本地项目目录（Windows）执行：

```bash
# 如果还没有初始化Git仓库
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit: HVAC industry website"

# 添加远程仓库（替换 YOUR_USERNAME 为你的GitHub用户名）
git remote add origin https://github.com/YOUR_USERNAME/wangzhan.git

# 推送代码
git branch -M main
git push -u origin main
```

### 第三步：云服务器部署

SSH连接到你的腾讯云服务器，执行以下命令：

```bash

        proxy_http_version 1.1;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;

systemctl reload nginx
systemctl enable nginx
```

## PM2 进程管理详解

本项目使用 PM2 管理两个核心进程：
- `wangzhan-web` - Next.js 网站服务（运行在 3000 端口）
- `task-scheduler` - 定时任务调度器（执行 AI 内容生成和微信同步）

### PM2 常用命令

#### 1. 查看进程状态

```bash
# 查看所有进程状态
pm2 status

# 输出示例：
# ┌────┬────────────────────┬─────────────┬─────────┬─────────┬──────────┬────────┬──────┬───────────┬──────────┬──────────┬──────────┬──────────┐
# │ id │ name               │ namespace   │ version │ mode    │ pid      │ uptime │ ↺    │ status    │ cpu      │ mem      │ user     │ watching │
# ├────┼────────────────────┼─────────────┼─────────┼─────────┼──────────┼────────┼──────┼───────────┼──────────┼──────────┼──────────┼──────────┤
# │ 0  │ task-scheduler     │ default     │ 1.0.0   │ fork    │ 3617042  │ 3h     │ 0    │ online    │ 0%       │ 63.2mb   │ root     │ disabled │
# │ 2  │ wangzhan-web       │ default     │ N/A     │ fork    │ 3666848  │ 7s     │ 1    │ online    │ 0%       │ 59.2mb   │ root     │ disabled │
# └────┴────────────────────┴─────────────┴─────────┴─────────┴──────────┴────────┴──────┴───────────┴──────────┴──────────┴──────────┴──────────┘
```

**字段说明**：
- `id` - 进程ID
- `name` - 进程名称
- `uptime` - 运行时间
- `status` - 状态（online=运行中，stopped=已停止，errored=错误）
- `cpu` - CPU使用率
- `mem` - 内存使用
- `↺` - 重启次数

#### 2. 启动进程

```bash
cd /root/wangzhan

# 启动网站服务（首次启动）
pm2 start npm --name "wangzhan-web" -- start

# 启动定时任务调度器（首次启动）
pm2 start scripts/task-scheduler.js --name "task-scheduler"

# 保存配置，确保重启后自动启动
pm2 save
```

#### 3. 重启进程

```bash
# 重启网站（代码更新后使用）
pm2 restart wangzhan-web

# 重启定时任务
pm2 restart task-scheduler

# 重启所有进程
pm2 restart all

# 零停机重启（推荐用于生产环境）
pm2 reload wangzhan-web
```

#### 4. 停止进程

```bash
# 停止网站
pm2 stop wangzhan-web

# 停止定时任务
pm2 stop task-scheduler

# 停止所有
pm2 stop all
```

#### 5. 删除进程

```bash
# 删除网站进程（从PM2列表中移除）
pm2 delete wangzhan-web

# 删除定时任务
pm2 delete task-scheduler

# 删除所有
pm2 delete all
```

#### 6. 查看日志

```bash
# 查看所有日志
pm2 logs

# 查看网站日志（实时）
pm2 logs wangzhan-web

# 查看定时任务日志
pm2 logs task-scheduler

# 查看最近100行日志
pm2 logs wangzhan-web --lines 100

# 查看错误日志
pm2 logs wangzhan-web --err

# 清空日志
pm2 flush
```

#### 7. 监控和诊断

```bash
# 实时监控（CPU、内存等）
pm2 monit

# 显示详细信息
pm2 show wangzhan-web

# 查看进程配置
pm2 config wangzhan-web
```

### 故障处理指南

#### 场景1：网站无法访问

**排查步骤**：
```bash
# 1. 检查进程是否在运行
pm2 status

# 2. 查看网站日志
pm2 logs wangzhan-web --lines 50

# 3. 检查3000端口是否在监听
netstat -tlnp | grep 3000

# 4. 本地测试访问
curl http://localhost:3000

# 5. 如果进程停止，重新启动
pm2 restart wangzhan-web
```

**常见问题**：
- 如果显示 `stopped`：执行 `pm2 restart wangzhan-web`
- 如果显示 `errored`：查看日志 `pm2 logs wangzhan-web`，修复错误后重启
- 如果端口被占用：`netstat -tlnp | grep 3000` 查看占用进程并杀死

#### 场景2：定时任务不执行

**排查步骤**：
```bash
# 1. 检查调度器是否在运行
pm2 status

# 2. 查看调度器日志
pm2 logs task-scheduler --lines 100

# 3. 检查任务文件是否存在
ls -la /root/wangzhan/content/tasks/

# 4. 查看任务文件内容
cat /root/wangzhan/content/tasks/*.json

# 5. 重启调度器
pm2 restart task-scheduler
```

**常见问题**：
- 如果调度器停止：执行 `pm2 restart task-scheduler`
- 如果任务文件丢失：需要在后台管理界面重新创建任务
- 如果任务状态不是 active：修改任务文件中的 `status` 为 `active`

#### 场景3：新闻生成成功但不显示

**排查步骤**：
```bash
# 1. 检查新闻文件是否存在
ls -la /root/wangzhan/content/news/

# 2. 检查文件内容
cat /root/wangzhan/content/news/2026-*.md | head -20

# 3. 检查文件权限
ls -la /root/wangzhan/content/news/*.md

# 4. 重启网站服务（清除缓存）
pm2 restart wangzhan-web

# 5. 查看网站日志
pm2 logs wangzhan-web --lines 50
```

**注意**：Next.js 会缓存文件列表，新生成的文件需要重启 `wangzhan-web` 才能显示。

#### 场景4：内存占用过高

```bash
# 查看内存使用情况
pm2 status

# 重启服务释放内存
pm2 restart wangzhan-web
pm2 restart task-scheduler
```

#### 场景5：PM2 开机自启失效

```bash
# 重新设置开机自启
pm2 startup

# 保存当前进程列表
pm2 save
```

## 注意事项

1. **备案问题**：使用腾讯云中国境内服务器，需要完成ICP备案才能正常访问
2. **环境变量**：`.env` 文件包含敏感信息，不要提交到Git仓库
3. **防火墙**：确保服务器防火墙开放80端口
4. **权限**：确保Nginx有权限访问 `/root/wangzhan/` 目录下的文件
